## 3. 조직 설계 포인트

**본부 분리 이유**
- **경영지원본부**: 전사 공통 지원 기능(인사·재무·총무·법무·IT운영)을 묶되, "부 → 팀" 2단계로 단순화하여 별도 본부장 없이 부서장이 실무 최종 승인권자 역할 수행(결재 신속성 확보).
- **AI사업본부 / 플랫폼사업본부 분리**: AI사업본부는 "제품/모델" 중심, 플랫폼사업본부는 "인프라/SaaS 플랫폼" 중심으로 목적함수가 달라 자원배분 우선순위 충돌을 피하기 위해 분리. 인프라 공동사용 지점(GPU/클라우드)은 매트릭스 협업으로 보완.
- **영업본부**: Enterprise·공공·고객성공을 한 지휘체계 아래 두어 "수주 이전(영업)"과 "수주 이후(고객성공)"의 고객 경험 연속성 확보.
- **전략기획본부**: 중장기 전략·신사업·PMO를 하나로 묶어 각 사업본부 개별 최적화가 전사 전략과 어긋나지 않도록 조율하는 컨트롤타워.

**협업 구조**
- AI사업본부 **MLOps팀**과 플랫폼사업본부 **클라우드부**는 점선보고(매트릭스) 관계. 공식 소속은 AI사업본부이나 GPU·클라우드 리소스 운영은 클라우드부와 공동 수행 → "신청 조직 = 실사용 조직"이 일치하지 않는 예외 케이스 발생, 비용귀속·정산 프로세스 설계 시 별도 규정 필요.

**의사결정 구조**
- CEO 직속 **경영회의체**(본부장급 협의체)가 전사 의사결정 중심.
- **감사실**은 보고라인상 CEO 직속이나, 경영진 견제 목적의 독립 기능으로 운영(감사 대상에 경영진 의사결정도 포함).
- 팀장 → 부서장 → 본부장 → CEO의 단계적 위임전결 구조가 기본. 본부장이 없는 경영지원본부는 부서장 전결 또는 CEO 직접 결재로 대체.

**조직 운영 특징**
- 사업본부별 독립 손익(BU) 운영으로 본부장에게 예산·인력 운용 재량 부여, 전사 공통 리스크(법무·보안·감사)는 경영지원본부·감사실이 횡단적으로 통제.
